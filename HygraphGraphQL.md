# Hygraph GraphQL — API Playground Setup

All schema changes go through a single `submitBatchChanges` mutation.
Use the **Management API** playground in Hygraph.

---

## Step 1 — Create model + all fields (one shot)

Paste the mutation and variables separately in the playground.

### Mutation

```graphql
mutation CreateBlogPostSchema($data: BatchMigrationInput!) {
  submitBatchChanges(data: $data) {
    migration {
      id
      status
      errors
      createdAt
      finishedAt
    }
  }
}
```

### Variables

```json
{
  "data": {
    "environmentId": "24240d72604845e5a9506f93bf005776",
    "name": "add-blog-post-model",
    "changes": [

          "apiId": "BlogPost",
          "apiIdPlural": "BlogPosts",

```

Check the response — `status` should be `SUCCESS`. If it says `QUEUED` or `RUNNING`, re-run the status check below.

## Step 2 — Check migration status (if needed)

If the response came back with status `QUEUED` or `RUNNING`, paste the `id` from the response here:

```graphql
query MigrationStatus {
  viewer {
    ... on UserViewer {
      projects {
        environments {
          migration(id: "PASTE_MIGRATION_ID_HERE") {
            id
            status
            errors
            finishedAt
          }
        }
      }
    }
  }
}
```

## Step 3 — Seed content (Content API Playground)

Switch the playground dropdown to **Content API**, then run these.

### Create a post

```graphql
mutation {
  createBlogPost(data: {
    blogTitle: "Welcome to Shinobi Blog"
    blogPostSlug: "welcome-to-shinobi-blog"
    blogPostContent: {
      children: [
        {
          type: "paragraph"
          children: [{ text: "This is your first post on the Shinobi Blog." }]
        }
      ]
    }
  }) {
    id
    blogTitle
  }
}
```

### Publish the post

Replace `YOUR_ID` with the `id` returned above:

```graphql
mutation {
  publishBlogPost(where: { id: "YOUR_ID" }) {
    id
    blogTitle
  }
}
```

Repeat create + publish for each post you want.

## Reference — fields expected by the app

| Field | Type | Notes |
|---|---|---|
| `blogTitle` | String | Required, used as display title |
| `blogPostSlug` | String | Required, unique — used in URLs |
| `blogPostContent` | Rich Text | Main body content |
| `createdAt` | DateTime | Auto-generated by Hygraph |
| `createdBy` | Author ref | Auto-generated by Hygraph |
